# Future OTA Integration Roadmap

## Phase A — Sandbox adapters

- Booking.com Demand API
- Expedia Rapid
- Skyscanner affiliate/partner APIs

Each lands behind the existing adapter contract with simulated→sandbox→production flags.

## Phase B — Direct / specialised inventory

- Airline NDC / direct programmes
- Hotel group APIs
- Car-hire affiliates
- Activity marketplaces
- Insurance + eSIM partners

## Phase C — Commercial operations

- Conversion postbacks into the attribution module
- Partner-branded landings (already scaffolded)
- SLA monitoring on the operations dashboard
- Contractual disclosure localisation

## Non-negotiables

- Ranking remains commission-independent
- No scraping
- No payments inside Travel Buddy until a dedicated payments phase
- Partner centre metrics stay null/honest until production telemetry exists
